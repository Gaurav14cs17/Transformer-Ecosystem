# LLaVA — Visual Instruction Tuning

## Overview

LLaVA (Large Language and Vision Assistant) is a multimodal model that connects a CLIP vision encoder to an LLM (Vicuna/LLaMA) via a simple linear projection, then fine-tunes on GPT-4-generated visual instruction-following data. It demonstrates that visual instruction tuning — training on language-image instruction-following data — endows LLMs with multimodal chat capabilities comparable to GPT-4V on many tasks.

## Architecture & Mechanism

LLaVA uses a simple three-component architecture:

1. **Vision encoder**: CLIP ViT-L/14 (frozen) encodes images into visual feature tokens.
2. **Projection layer**: A trainable linear layer (or MLP in later versions) maps visual tokens into the LLM's word embedding space.
3. **LLM**: Vicuna-13B (fine-tuned LLaMA) generates text responses conditioned on visual and text tokens.
4. **Two-stage training**:
   - **Pre-training**: Align vision and language via image captioning on CC3M (595K pairs). Only the projection layer is trained.
   - **Fine-tuning**: Visual instruction tuning on 158K GPT-4-generated instruction-following conversations. Both projection and LLM are fine-tuned.

## Key Innovations

- **Visual instruction tuning**: First to use GPT-4 to generate multimodal instruction-following data for training.
- **Simple architecture**: Linear projection between CLIP and LLM — surprisingly effective.
- **GPT-4-generated data**: High-quality training data generated by prompting GPT-4 with image captions and bounding boxes.
- **Multimodal chat**: Natural conversational interface for visual understanding.
- **Open-source**: Fully open weights, data, and training code, catalyzing the open VLM ecosystem.
