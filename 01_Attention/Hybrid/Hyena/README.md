# Hyena (Long Convolution Alternative to Attention)

A subquadratic alternative to attention that uses long convolutions and data-controlled gating to achieve attention-quality performance without the quadratic cost.

## Architecture

Hyena replaces attention with a hierarchy of long convolutions and element-wise gating:

```
Hyena operator of order N:
  x₁, x₂, ..., x_N = Linear projections of input (N parallel projections)
  h₁, h₂, ..., h_{N-1} = Learned long convolution filters

  y₁ = x₁
  y₂ = (h₁ * y₁) ⊙ x₂        (* = long convolution, ⊙ = element-wise gate)
  y₃ = (h₂ * y₂) ⊙ x₃
  ...
  output = y_N

Long convolutions computed via FFT: O(n log n) complexity
Filter parameterization: Implicit neural network or explicit decay
```

## Key Innovations

- **Subquadratic**: O(n log n) complexity via FFT-based convolution
- **Data-controlled**: Gating with input projections provides data-dependent processing
- **No attention matrix**: Eliminates the n×n attention matrix entirely
- **Long convolutions**: Captures long-range dependencies through learned convolutional filters
- **Implicit filter parameterization**: Filters generated by a small neural network (avoids storing full filter)
- **Drop-in replacement**: Same input/output interface as standard attention

## Performance

- Matches attention quality on language modeling up to 100B+ FLOPs scale
- 100× throughput improvement over attention at sequence length 64K
- Competitive with S4, H3, and other state-space alternatives
- Scales well on the Long Range Arena benchmark

## Usage

Hyena is used for:
- Long-sequence language modeling
- Tasks requiring subquadratic scaling with sequence length
- Audio and signal processing (natural fit for convolutions)
- As a building block in hybrid architectures (e.g., StripedHyena)
